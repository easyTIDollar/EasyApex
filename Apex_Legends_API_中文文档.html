简介

欢迎使用非官方 Apex Legends API，为您提供便捷的玩家数据与游戏数据接口服务。

  • 本 API 为非官方产品，不提供任何担保，不保证正常运行时间
  • 使用前必须在 开发者门户 注册 API Key
  • 每人/每项目仅限一个 API Key，违规可能导致 Key 被删除
  • 适用公平使用原则，请勿进行对 API 有害的操作
  • 如有疑问，可在 Discord 上私信 hugo#0069
🔑

身份认证

所有 API 请求均需提供有效的 API Key，支持两种方式传递。

方式一：Authorization 请求头

bash
# 使用 shell 时在请求头中携带 API Key curl "https://api.apexlegendsstatus.com/" -H "Authorization: YOUR_API_KEY" # 或者在 GET 参数中传递 curl "https://api.apexlegendsstatus.com/?auth=YOUR_API_KEY"

方式二：Query Parameter

url
https://api.apexlegendsstatus.com/bridge?auth=YOUR_API_KEY&player=PlayerName&platform=PC
⚠
重要：请将 YOUR_API_KEY 替换为您的真实 API Key。
⏱

速率限制

  • 首次创建 API Key 时，限制为 5 次请求/秒
  • 可通过绑定 Discord 账号或在 Discord 开票据申请提高限制
  • 所有 API 共享此速率限制
  • 响应头 X-Current-Rate 可查看当前速率
🔨

Webhook

  • 目前仅 比赛历史 API 支持 Webhook
  • 必须在开发者门户配置 Webhook URL
  • Webhook 使用的 User-Agent 固定为 ApexAPI Webhook/0.1
  • 服务器必须在收到 POST 后 3 秒内 完成处理，否则丢弃

玩家统计数据

通过玩家名称或 UID 获取详细的 Apex Legends 游戏统计数据。

按名称查询

通过玩家名称和平台获取玩家统计数据。PC 玩家需使用 Origin 账号名称。

GET
https://api.apexlegendsstatus.com/bridge?auth=YOUR_API_KEY&player=PLAYER_NAME&platform=PLATFORM

查询参数

参数必填描述
player是玩家用户名
platform是PC（Origin/Steam）、PS4（PS4/5）、X1（Xbox）
version否API 版本（1、2、4 或 5），默认 5，其他版本已不再活跃支持
enableClubsBeta否true/false。获取玩家公会数据（测试中）
skipRank否任意值。不返回排位统计数据
merge否任意值。合并相同类型的追踪器数据
removeMerged否任意值。移除已被合并的追踪器

按 UID 查询

通过玩家 UID 和平台获取玩家统计数据。

GET
https://api.apexlegendsstatus.com/bridge?auth=YOUR_API_KEY&uid=PLAYER_UID&platform=PLATFORM

查询参数

参数必填描述
uid是玩家 UID
platform是PC、PS4、X1、SWITCH
version否API 版本（1、2、4 或 5），默认 5
enableClubsBeta否获取公会数据（测试中）
skipRank否不返回排位数据
merge否合并追踪器数据
removeMerged否移除已被合并的追踪器

比赛历史

新版比赛历史 API

获取指定玩家的比赛历史，返回最近 20 场比赛数据。

GET
https://api.apexlegendsstatus.com/games?auth=YOUR_API_KEY&uid=PLAYER_UID

查询参数

参数必填描述
uid是玩家 UID
mode否BATTLE_ROYALE、ARENAS、UNKNOWN
start否开始 epoch 时间戳（整数）
end否结束 epoch 时间戳（整数）
limit否最多返回场次数（整数）

返回字段

字段类型描述
idstring比赛唯一 ID
platformstring平台
match_lengthint比赛持续时间（秒）
start/endint开始/结束 epoch 时间戳
mapstring地图名称
modestring游戏模式
seasonint赛季编号
match_typestring比赛类型（PUB/RANKED/PUB_PRACTICE）
rankedboolean是否排位赛
is_duo/is_solo/is_trioboolean双排/单排/三排
playersarray玩家数组

players 数组字段

字段类型描述
uidstring玩家 UID
namestring玩家名称
rankstring段位
kills/assistsint击杀/助攻
damageint造成伤害
team_positionint队伍排名（1-12）
legendstring传奇角色
levelint传奇等级
rank_scoreint排位分数
team_placementint队伍最终排名
team_damageint队伍总伤害
club_tagstring公会标签
📡
白标要求：白标用户需在页面添加指向 apexlegendsstatus.com 的链接，文案为 "Data provided by Apex Legends Status"。

旧版比赛历史 API

旧版端点，同时支持查询比赛历史和追踪玩家列表管理。

GET
https://api.apexlegendsstatus.com/bridge?auth=YOUR_API_KEY&uid=PLAYER_UID&platform=PLATFORM&history=1&action=ACTION
参数必填描述
uid是玩家 UID
platform是PC、PS4、X1、SWITCH
history是比赛历史模式，值必须为 1
action是info、get、delete、add
⚠
注意：此 API 已不再活跃维护，推荐使用 新版比赛历史 API。
🏅

排行榜

获取各传奇的排行榜数据，最多返回前 500 名玩家。

GET
https://api.apexlegendsstatus.com/leaderboard?auth=YOUR_API_KEY&legend=LEGEND&key=KEY&platform=PLATFORM
参数必填描述
platform是PC、X1、PS4、SWITCH、ANY
legend是传奇名称（首字母大写），Global 获取全局数据
key否追踪器键名。不传返回所有可用键名列表
🗺

地图轮换

返回当前和即将到来的 Battle Royale 及 Arenas 模式下的轮换地图。

GET
https://api.apexlegendsstatus.com/maprotation?auth=YOUR_API_KEY
参数必填描述
version否1 仅 Battle Royale 匹配地图；2 所有模式
💡
不传 version 或传 1 仅返回 Battle Royale 地图；传 2 可获取包含 Arenas 的完整数据。
👑

猎杀者（Predator）

返回各平台达到猎杀者段位所需的 RP/AP 及大师玩家数量。

GET
https://api.apexlegendsstatus.com/predator?auth=YOUR_API_KEY
字段类型描述
platformstring平台（PC、PS4、X1、SWITCH）
rank_scoreint猎杀者所需排名分数（RP）
arena_scoreint猎杀者所需竞技场分数（AP）
masters_countint大师段位玩家数量

服务器状态

显示各区域服务器状态，与 apexlegendsstatus.com 一致。

GET
https://api.apexlegendsstatus.com/servers?auth=YOUR_API_KEY
📡
白标要求：需包含指向 apexlegendsstatus.com 的链接或注明 "Data from apexlegendsstatus.com"。

Origin 用户名查询

仅获取玩家的 Origin UID，不返回统计数据。

GET
https://api.apexlegendsstatus.com/origin?auth=YOUR_API_KEY&player=PLAYER_NAME
参数必填描述
player是玩家名称

名称转 UID

根据玩家名称返回 UID，支持 PS、Xbox 和 Origin。

GET
https://api.apexlegendsstatus.com/nametouid?auth=YOUR_API_KEY&player=PLAYER_NAME&platform=PLATFORM
参数必填描述
player是玩家名称
platform是PS4、X1、PC
